# Support

This document describes how to get help for SentientWave Automata Community Edition.

## Community Support

- Open a GitHub issue for bugs and feature requests.
- Use issue templates to provide reproducible context.
- Include version from `VERSION`, deployment mode, and logs.

## Security Issues

Do not report vulnerabilities in public issues.

- Email: `security@sentientwave.ai`
- Policy: see [SECURITY.md](SECURITY.md)

## What To Include In Support Requests

- Automata version (`VERSION`)
- Deployment type (local dev, all-in-one container, custom)
- Matrix homeserver URL and relevant component versions
- Reproduction steps and expected versus actual behavior
- Relevant logs and timestamps

## Response Expectations

- Community support is best-effort.
- Critical security reports are prioritized through the private security channel.
- Enterprise customers should use contracted support channels.

## Out Of Scope

- Support for modified forks not reproducible on current `main`
- Help with third-party outages unrelated to Automata code
- Recovery from unsupported manual data mutations
